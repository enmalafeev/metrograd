## О проекте

«Метроград» — демо мобильного 3D-симулятора поезда метро в стиле пиксель-арт на **Godot 4** (Forward+). Тестируется на Mac (Godot ставится через `brew install --cask godot`). Комментарии и UI — на русском.

## Команды

```bash
make run                 # запустить проект (godot --path .)
python3 tools/gen_textures.py   # пересобрать пиксельные текстуры в assets/textures/ (нужен Pillow)
```

Синтаксическая проверка скрипта без запуска редактора:

```bash
godot --headless --check-only --script scripts/world.gd
```

Тест-фреймворка в проекте нет; проверка — запуск демо и `--check-only` по скриптам.

## Архитектура

Одна сцена `scenes/main.tscn`. 3D-мир рендерится в `SubViewport` 640×360 (`own_world_3d`) и выводится на весь экран через `TextureRect` с фильтрацией Nearest — так получается чёткая «пиксельная» картинка при любом разрешении. `main.gd` (корневой `Node`) держит и `SubViewport`, и `HUD` (`CanvasLayer`).

Три узла внутри мира, каждый со своим скриптом, инициализируются в `main.gd._ready()` в строгом порядке:

1. **`world.gd`** (`World`) — процедурно строит всю геометрию в `_ready()`. Знать про его модель координат обязательно перед правками геометрии или маршрута:
   - Два параллельных пути вдоль оси Z на `x = ±T`. Маршрут — **одна замкнутая кривая** (`Curve3D`): прямой участок по внешнему пути (`x=-T`) → полукруглая петля на дальней конечной → обратно по внутреннему пути (`x=+T`) → петля на ближней конечной, замыкающая кольцо. Направление движения задаётся геометрией кривой (петлями), а не поездом.
   - Станции размещаются по накопленным перегонам `GAPS` (переменной длины), см. `_compute_layout()`/`station_z()`.
   - `stops` — отсортированный по кривой список остановок: по **две** на станцию (внешняя и внутренняя платформа), с полями `offset` (позиция вдоль кривой), `x`, `idx`, `terminal`. Это контракт, который читают `main.gd` и `passengers.gd`.
   - Вся геометрия — примитивы `BoxMesh` через хелпер `_box()`; материалы кэшируются в `_mats` через `_mat()`.

2. **`train.gd`** (`Train`) — состав из `CARS` вагонов. Движется по кривой по скалярной дистанции `progress` (центр состава). **Каждый вагон ставится на кривую отдельно** по своему смещению и ориентируется по касательной (`look_at` на точку чуть впереди) — поэтому состав изгибается на петлях. Кинематика тяги/торможения/выбега — в `_physics_process()` через `throttle` (-1..1) и `doors_open`. Головной вагон несёт кабину, фары (`SpotLight3D`) и **четыре камеры** — кабина/салон/снаружи/спереди (перечисление `View`, переключение `set_view(index)`). Геометрия вагона и салона тоже собирается из боксов в коде.

3. **`passengers.gd`** (`Passengers`) — блочные фигурки на платформах. `board_and_alight(idx, side)` вызывается из `main.gd` при открытии дверей; ходьба реализована твинами. `_waiting` индексируется ключом `"idx:side"`.

**`main.gd`** — HUD, ввод и логика маршрута. В `_process()` читает рычаг/клавиши и прокидывает `throttle`/`doors_open` в поезд. Знаковая дистанция до следующей остановки — `_remaining()` (через `fposmod` по длине кривой); `_aligned()` проверяет точную остановку у центра платформы. `next_stop` продвигается по кольцу `world.stops` при закрытии дверей.

**`cab_controls.gd`** — дискретный контроллер машиниста (`Control`, отрисовка в `_draw()`): фиксированные позиции Х3..0..Т3, поле `value` (-1..1) читает `main.gd`. Дублируется клавишами (см. ниже).

## Ввод

Действия заданы в `project.godot`: `throttle_up` (W), `throttle_down` (S), `doors` (пробел). На тач/мыши — рычаг контроллера и экранные кнопки «ДВЕРИ» / «В САЛОН».

## Соглашения

- Ассеты (мир, вагоны, пассажиры) строятся **процедурно из `BoxMesh` в коде**, а не в `.tscn`. Правки внешнего вида и геометрии идут в `.gd`, не в сцене.
- Числовые константы вынесены в начало каждого скрипта (`const`), с комментарием и единицами (обычно метры). Меняй их, а не «магические числа» по коду.
- Текстуры — маленькие тайлящиеся PNG из `gen_textures.py`, с фильтрацией Nearest (пиксель-арт). После правки генератора пересобери и закоммить PNG.
